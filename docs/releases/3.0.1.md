# InnoNetworkProtobuf 3.0.1 Release Notes

## Summary

`InnoNetworkProtobuf` 3.0.1 is the first public protobuf adapter package for the `InnoNetwork` family.

It keeps Protocol Buffers request serialization and response decoding outside the core package, so consumers that only need JSON, form, download, or websocket functionality do not resolve `swift-protobuf`.

## Highlights

- Ships `ProtobufAPIDefinition`, `ProtobufNetworkClient`, `ProtobufEmptyResponse`, and `HTTPEmptyResponseMessage`
- Adds documentation, contract checks, protobuf doc smoke validation, and consumer smoke validation
- Pins the core dependency to `InnoNetwork` `3.0.1`

## Upgrade Notes

- Add `InnoNetworkProtobuf` alongside `InnoNetwork`
- Keep both packages on the same `3.x` release line
- GET requests with protobuf parameters remain unsupported because protobuf payloads are body-only

## Validation

Release validation for 3.0.1 includes:

- `swift test`
- docs / contract sync
- documentation smoke build and run
- external consumer smoke build
